# BayCuci 可用性调研报告（2026-05-17）

## 1) 调研目标
- 评估两类用户对当前 BayCuci Web App 的适用性：
  - 精致上班族（效率导向、移动端高频操作）
  - 高科技老人家（愿意用手机应用，但对复杂步骤敏感）
- 输出“方便性 / 简洁性”评分（满分 10 分）与可执行优化建议。

## 2) 测试范围与方法
- 范围：预约主流程、表单校验、提交成功、异常反馈、语言切换、客服触达、邀请分享。
- 方法：
  1. 自动化质量基线：`npm test`、`npm run build`、`npm run audit:release`
  2. 后端联通验证：`smoke:rpc` + 双画像真实 RPC 提交
  3. 关键异常验证：手机号、日期、周末限制、地址“其他”分支
  4. 任务导向可用性走查：按用户目标逐步评估点击路径与认知负担

## 4) 用户画像与任务设计

### A. 精致上班族
- 特征：时间碎片化、希望 1 分钟内完成预约、偏好少输入。
- 核心任务：
  1. 选择区域与车位
  2. 选择包月/单次
  3. 选择日期与时段
  4. 快速提交并确认后续沟通方式（WhatsApp）

### B. 高科技老人家
- 特征：能用智能手机，但对“隐藏规则”和“术语”敏感。
- 核心任务：
  1. 理解服务是否可信（主体、客服、隐私）
  2. 在错误输入时看懂提示并改正
  3. 成功提交并确认下一步

## 5) 测试发现（按任务）

### 5.1 共同正向体验
- 表单结构清晰，字段顺序符合“从身份到车位再到时间”的思路。
- 支持 WhatsApp 帮助入口，降低提交前犹豫。
- “Booking preview”在提交前给出摘要，降低填错风险。
- 成功页包含预约摘要与分享操作，流程闭环完整。

### 5.2 精致上班族视角
- 优点：
  - 操作路径短，主流程单页完成，无跳转中断。
  - 手机号自动标准化（0/60/+60/0060）减少格式负担。
  - 常用区域预置选项，减少输入时间。
- 阻力：
  - 仍需手动输入停车楼层和车位号，无法复用历史记录。
  - 时段命名（`night_1` 等映射）在前后端旧约束并存时存在理解/维护成本。

### 5.3 高科技老人家视角
- 优点：
  - 标签文案直白，必填校验会明确指出字段。
  - “Trusted service”信息块对建立信任有帮助。
  - 多语言切换可用（English/中文/BM）。
- 阻力：
  - 周末不可选是业务规则，但用户只有在提交/校验后才完全感知。
  - “Other area”触发二次输入，对不熟悉表单分支的用户有轻微困惑。
  - 邀请返现模块在成功页出现，部分老人用户会分心，不利于“确认预约完成”主目标。

## 6) 评分（满分 10）

### 精致上班族
- 方便性：**8.7 / 10**
- 简洁性：**8.3 / 10**
- 评分依据：流程短、反馈快、可单页完成；但车位信息重复输入和 slot 兼容逻辑拉低极限效率。

### 高科技老人家
- 方便性：**7.9 / 10**
- 简洁性：**7.4 / 10**
- 评分依据：整体可完成，但“规则后置感知 + 成功页次要任务干扰”对认知负担有影响。

## 7) 关键问题优先级（落地导向）
- P0：统一数据库 slot 约束与前端 slot 枚举，去掉隐式兼容分支，避免线上语义漂移。
- P1：把“仅工作日可预约”前置到日期控件附近（说明文 + 禁选提示），减少提交后挫败。
- P1：新增“最近一次车位信息复用”按钮（楼层/车位一键带入）。
- P2：成功页默认聚焦“预约成功与下一步”，将邀请返现折叠为次级入口。
- P2：在 `Other area` 被选中时加一行即时提示（示例格式），降低二次输入失败率。

## 8) 结论
- 当前版本已具备可上线的基础可用性与稳定性，特别适合“精致上班族”快速预约场景。
- 面向“高科技老人家”再提升 1~2 个版本可明显提高转化：优先做规则前置、成功页减负、历史信息复用。
